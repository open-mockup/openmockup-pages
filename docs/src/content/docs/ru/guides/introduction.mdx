title: Введение
description: Что такое OpenMockup, для кого он, и чем он не является.
---

OpenMockup — декларативный инструмент для быстрых UI-мокапов и
пользовательских флоу. Вы описываете экраны на компактном языке
разметки; рендерер превращает их в живые интерактивные превью. Флоу
связывает экраны в проходимый прототип, а версии можно сравнивать
визуально.

## Для кого это

- **Дизайнеры**, которым нужен прототип быстрее, чем в векторном
  редакторе, с настоящими компонентами и реальной интерактивностью.
- **Продакт-менеджеры**, которые хотят передавать замысел без
  пиксель-перфектных мокапов, которые потом путают с готовым дизайном.
- **Инженеры**, которые набрасывают форму UI до того, как закоммитятся
  в конкретный стек.

## Чем это не является

- **Не продакшен UI-фреймворк.** Мокапы намеренно приблизительные.
  Не нужно их шипать.
- **Не замена Figma.** Нет пиксель-перфекта, нет дизайн-токенов,
  нет auto-layout в смысле дизайн-инструмента.
- **Не кодогенератор.** Разметка предназначена для мокапов; вывод не
  переносится в реальное приложение как есть.

## Из чего это состоит

1. **Язык** — небольшой декларативный DSL для композиции компонентов.
2. **Библиотека компонентов** — примитивы (кнопки, лейауты, поля
   ввода), на которые ссылается язык.
3. **Flow** — экраны, связанные действиями, с историей версий.
4. **AI-помощник** — генерация разметки мокапов по промпту.
5. **Интеграции** — плагин для VS Code расширяет поверхность
   (сегодня там же живёт визуальный diff флоу).
